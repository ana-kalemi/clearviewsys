Declaring AML Rules

This section shows some screenshots on how AML Rules can now be declared by compliace officers. This module is only available in version 3.610 of CurrencyXchanger and higher.

The AML Rules Declaration module

This new module can be accessed from the pull down menu in the bottom the modules.

Zoom

AML Rules can be defined by administrators and compliance officers only. This module is designed from the ground up to allow custom AML rules to be defined by different companies and hence adhering to the policies and guidelines set forth in a company's compliance regime (AML manual).

Screenshot 1: The declaration of the $500 Rule

Zoom

Declaring the $500 Rule:   Require information government issued ID information must be type in at $500 including: country of id, type of ID, id #, name, address, exp date (not required), phone #

Screenshot 2: Declaration of the $1000 Rule

Zoom

$1000 Rule: Additionally, require that consumer ID must be scanned into system and teller must check boxes indicating that the ID is a government issued ID and that the ID is still valid (not expired)

Screenshot 3: Declaration of the $3000 Rule

Zoom

$3000 Rule: Additionally, require that we obtain a statement from the consumer as to their “occupation” and that this must stored in the system.  If the consumer has provided a US ID#, we must acquire a Social Security # or a statement that will provide “None”.

Screenshot 4: Declaration of the $10,000 Rule

Zoom

$10,000 Rule:  Acquire sufficient information for filing of CTR, including any backup information, and electronically file the CTR if possible.  Five or six standard types plus an other + scanned documentation depending on the source of funds all: bank statement, tax return, pay stubs, legal settlement, contract of sales, other - (need typed description).  Attached scan of items.  Teller must check box indicating that documents are current.  

Activating the Rule Based System

Make sure you login as administrator to activate the AML Rule Structure.

In order for the AML rules to trigger, you have to make sure to check the box shown in the above screenshot.

Other AML & Compliance configurations in the system

Under the Compliance & Warnings, make sure you setup threshholds and activate OFAC checks.
